⚠ Divergências
  • Naming: Java usa PascalCase nos métodos (GetDeviceName) em vez de camelCase como todos os outros managers Java — inconsistência a corrigir.
  • Bulk DTOs: Go tem GetDeviceInfo(), GetCpuInfo(), GetOsInfo() que agregam os campos individuais num único DTO; Java não tem equivalente.
  • GetMetrics: Go devolve Metrics DTO limpo; Java devolve o proto raw MetricsInfoResponse.
  • Policies (Enable/Disable): Go devolve error; Java devolve boolean.
  • SendUpdate: Go recebe []byte + chunkSize; Java recebe File ou String path e faz chunking internamente.

⚠ Resumo das Divergências
Pontos a alinhar entre Go (source of truth) e Java
#ManagerProblemaSugestão
1 PowerManager Go devolve PowerResult; Java é void Uniformizar para void em Go ou devolver DTO em Java
2 GpioManager GetLevel → bool (Go) vs Level enum (Java). GetDirection igual. Preferir enum em ambos — mais expressivo e type-safe
3 GpioManager SetPwm → error (Go) vs PwmState (Java) Uniformizar — Go pode devolver PwmState
4 EthernetManager Go: SetConfig(DTO) — Java: setStaticConfig(...) + setDhcp() Alinhar Java para setConfig(EthernetConfig)
5 ✔ SpiManager Go tinha helpers Write/Read/WriteRead que Java não tinha ✔ Resolvido — API pública expõe só transfer()
6 ✔ I2CManager Go: ScanBus(bus) — Java: scan(bus) ✔ Resolvido — renomeado para scanBus(bus)
7 BluetoothManager Java tem GATT completo; Go não tem. Go tem GetConnectionState; Java não. Implementar GATT em Go; adicionar getConnectionState em Java
8 ✔ BluetoothManager Go: Connect/Disconnect — Java: connectDevice/disconnectDevice ✔ Resolvido — renomeado para connect/disconnect
9 DevelopmentManager Go LogFilter DTO — Java passa campos separados Criar LogFilter record em Java
10 UartManager Go tem Listen síncrono; Java não Aceitável — Java usa sempre callback async
11 ✔ SystemManager Java usava PascalCase (GetDeviceName) — diferente dos outros managers ✔ Resolvido — todos os métodos renomeados para camelCase (getDeviceName, etc.)
12 ✔ SystemManager Go tinha GetDeviceInfo() bulk — não existe no proto ✔ Resolvido — removido de Go; ambas as APIs expõem apenas os métodos individuais por RPC
13 SystemManager GetMetrics: Go devolve Metrics DTO; Java devolve proto raw MetricsInfoResponse Criar Metrics record em Java e converter
14 SystemManager (policies) Go AllowUntrustedApps() → error; Java → boolean (igual para Enable/Disable SSH, DevMode, RebootOnFailure) Uniformizar para void/error em ambos — o boolean de retorno é redundante
15 SystemManager (update) Go SendUpdate(ctx, []byte, chunkSize); Java Update(File) / Update(String) com chunking interno Adicionar overload sendUpdate(byte[]) em Java para consistência com Go
16 ✔ PackageManager Existia apenas em Java; métodos em PascalCase ✔ Resolvido — implementado em Go + métodos Java renomeados para camelCase
